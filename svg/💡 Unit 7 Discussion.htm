Recommendations for Project 7:


If you follow the study guide, they will have you using nested grids. This is annoying. Don't do it.

Follow the study guide to get the main grid figured out (header, navigation, body). Then switch to using Flexbox for the inner sections.

Nested Flexbox containers seem to be a lot easier for everyone to wrap their heads around than nested grids.


Chart.js:

Use the CDN (content delivery network) version of chart.js in your project. Don't download it and include it. (This is mentioned in the instructions.)

Here's how:

<script src="https://cdnjs.cloudflare.com/ajax/libs/Chart.js/3.3.2/chart.min.js"></script>

To resize your chart items, you'll need to use the aspectRatio option. The default is 2; adjust up or down from there as necessary, and test to see what works.

Resizing through CSS will not work, because chart.js is setting the size of the charts when the page loads, and overriding your CSS.


SVG Social Media Icons:

This is a very common component that you're going to run into a lot in your career, so it's a great one to learn now, and it's also very simple once you know how to do it.

The concept: Put your SVG inside a container element (div, etc.). In your CSS, the container element should have the background colour, border-radius, and height/width styling.

Style the SVG itself by setting a colour on the fill: of the SVG. (Other SVGs on other projects might need to be styled by targeting their path: instead, but for this project, it's all fill:.)

To create a circle in CSS, you need to start with a square. That means equal width and height. Then, add border-radius: 50%. Now it's a circle.

I recommend setting your sizes in rems for everything, but especially for these icons and their parent containers. That makes them scalable with the text and properly responsive. You should be able to write the CSS for these one time, in your base styles, and not need to adjust them in media queries for other sizes.


Other SVG Considerations:

In some cases, in some browsers, an SVG image might seem to not appear until you set a height/width on it. It's actually there, but it's just setting itself to such a huge height and width that you're looking through it or beside it. Set a width or a height, even temporarily, and it should come into view.


Exceeds Expectations:

LocalStorage is a built-in JavaScript feature. You can learn more about it here.

When it comes to saving the form fields, you will need to understand JSON.parse and when to (and when not to) use it. It's another built-in JavaScript feature.

In short: localStorage stores everything as a¬†string, which basically means it's storing it as text. But HTML checkboxes work on a true/false basis; they do not have a string value.

JSON.parse will take a string and return it as the correct type for JavaScript.

So while you need to use JSON.parse to parse the checkboxes, you do not need to use it to read the Time Zone stored value, because you want that as a string to replace the value of the select box.

Good question! I think the reality of it is that you'd most likely either be working with a small team¬†or you'd have some more experience under your belt if you were going to solo it.

So you'd either have someone to guide you a bit and collaborate with, or you wouldn't be doing all of this for the first few times ever. In either of those situations, I'd say maybe a week?

I'd think that at about 6 months into your actual career time, you'd have enough experience to have some common layout situations stuck in your head and know how to execute them, and you could probably knock this one out in a couple of full-time days. One day for HTML/CSS, and one for the JavaScript. Maybe a third day if you run into bugs or have to deal with crappy docs for the JS side of things.

The big problem with Treehouse is that you don't get much practice in once you learn something, so you're doing everything for the first or second time, and you haven't figured out common coding situations or been able to commit pieces of code or concepts to memory.

Once you're out of this learning situation and start getting experience with real projects, you'll be able to see how similar most of them are, and you'll get some mental shorthand going with the same types of things coming up over and over again.

If you look at Bootstrap's examples, you can see most of the web right there, broken down into page types or components. Those are the pieces that you'll eventually know how to build quickly, even if there's some variation in colour, type, spacing, etc., and then you'll just combine them into the final product.
